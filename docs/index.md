# Bitbucket Data Center Provider

The **bitbucketdc** provider manages resources on a self-hosted [Bitbucket Data Center](https://www.atlassian.com/software/bitbucket/enterprise) instance via the REST API.

## Authentication

The provider supports two authentication methods (use one only):

* **Personal Access Token** — recommended for automation.
* **Username / Password** — HTTP Basic Authentication.

Credentials can be supplied via provider attributes or environment variables.

## Example Usage

```terraform
# Authenticate using a Personal Access Token
provider "bitbucketdc" {
  base_url = "https://bitbucket.example.com"
  token    = var.bitbucket_token
}

# Or authenticate using username and password
provider "bitbucketdc" {
  base_url = "https://bitbucket.example.com"
  username = var.bitbucket_username
  password = var.bitbucket_password
}
```

## Schema

### Optional

- `base_url` (String) The base URL of your Bitbucket Data Center instance (e.g., https://bitbucket.example.com). Can also be set via the BITBUCKET_BASE_URL environment variable.
- `insecure_skip_verify` (Boolean) Skip TLS certificate verification. Can also be set via the BITBUCKET_INSECURE_SKIP_VERIFY environment variable. Not recommended for production use.
- `password` (String, Sensitive) Password for HTTP Basic Authentication. Can also be set via the BITBUCKET_PASSWORD environment variable. Must be used together with username.
- `timeout` (Number) Request timeout in seconds. Defaults to 30 seconds. Can also be set via the BITBUCKET_TIMEOUT environment variable.
- `token` (String, Sensitive) Personal Access Token for authentication. Can also be set via the BITBUCKET_TOKEN environment variable. Either token or username/password must be provided.
- `username` (String) Username for HTTP Basic Authentication. Can also be set via the BITBUCKET_USERNAME environment variable. Must be used together with password.
